# Purpose Atlas Source-UI Golden Witness Proposal

Status: proposal

This proposal incorporates the `purpose-atlas-v6-a2ui-ui-refactor.golden-witness.260618.zip`
improvements into `ui.git` as an A2UI exit-surface package.

## Claim

`packages/purpose-atlas-source-ui-golden` should be carried as a UI presentation
and witness package. It preserves the latest source UI shape, the cached
renderer improvements, and the A2UI v0.9 trust boundary.

It must not become the semantic authority for purpose, role, policy, guard, or
responsibility projection.

## Facts

- `AtlasSourceSurface` is registered in the default `ui.component.registry.v1`
  catalog as a `purpose_atlas` component.
- The imported package carries:
  - native A2UI v0.9 JSONL surface records in `public/a2ui/purpose-atlas.surface.jsonl`;
  - the source-UI component in `src/components/atlas-source-surface.js`;
  - the cached renderer in `src/ui/cached-atlas-renderer.js`;
  - source UI CSS in `src/styles/source-ui.css`;
  - golden source locks and witness scripts under `golden/` and `scripts/`;
  - evidence JSON/markdown from the golden witness run.
- `dist/` from the zip is intentionally not imported as source truth. It is a
  build product and must be regenerated by the package.
- The package lock was normalized to public npm registry tarball URLs for Nix
  adopter-readiness.

## Boundary

The package is an A2UI/UI exit surface:

```text
resolved a2ui.context.surface.v1
  -> adapter
  -> A2UI v0.9 createSurface/updateComponents/updateDataModel
  -> AtlasSourceSurface
  -> CachedAtlasRenderer
```

The browser package may render `/atlas`, dispatch allowlisted actions, and
record UI operations. It may not define canonical state, approval, fire, merge,
role policy, guard truth, or graph projection truth.

## Adapter Requirement

The next implementation step is a narrow adapter:

```text
raw/canonical/projection output
  -> resolved a2ui.context.surface.v1
  -> A2UI v0.9 message stream
```

That adapter must validate:

- catalog id and surface id;
- allowed DataModel roots: `/meta`, `/ui`, `/atlas`, `/inspector`, `/events`,
  `/operations`, `/toast`, `/runtime`;
- the 15 allowlisted Atlas actions;
- no executable payloads;
- no authority claims in UI records.

## Quarantine

`src/domain/atlas-engine.js`, `src/data/atlas-data.json`, and
`golden/source/model-core.js` are retained inside the proposal package only as
fixture/witness material from the zip. They are not promoted as canonical
domain, policy, or projection authority.

Acceptance requires a future adapter mode where `/atlas` is supplied by an
external resolved context surface rather than browser-side replay.

- Gen1-B verdict: `ADOPT` as source-UI-on-A2UI reference, with adapter required
